Переход на uv — это лучшее решение для экономии времени. Он делает всё то же самое, что и pip, но в 10–100 раз быстрее, потому что написан на Rust и эффективнее работает с кэшем.
Вот как быстро пересесть на него, сохранив привычку работать с requirements.txt и venv:
1. Установка uv
Сначала установите сам инструмент в систему (один раз):

* Windows (PowerShell):

powershell -c "irm https://astral.sh | iex"

* MacOS/Linux:

curl -LsSf https://astral.sh | sh


2. Работа с виртуальным окружением (venv)
Вместо стандартного python -m venv .venv используйте:

uv venv

Это создаст папку .venv практически мгновенно.
Для активации используйте те же команды, что и обычно:

* source .venv/bin/activate (Linux/Mac)
* .venv\Scripts\activate (Windows)

3. Установка из requirements.txt
Вместо pip install -r requirements.txt пишите:

uv pip install -r requirements.txt

Почему это круто:

* Скорость: Тот самый numpy, который у вас качался со скоростью 40 кБ/с, uv может подтянуть из своего глобального кэша за доли секунды (если он уже скачивался ранее).
* Параллельность: Он качает пакеты одновременно, а не по одному.

4. Одиночные пакеты
Если нужно просто добавить библиотеку:

uv pip install numpy

------------------------------
Совет: Если вы хотите вообще забыть про ручную активацию venv и requirements.txt, попробуйте команду uv init в новом проекте — она создаст современную структуру проекта, где зависимости управляются автоматически через файл pyproject.toml.
